# Shectory Wikipedia

Единый свод правил, документации и регламентов экосистемы **Shectory**. Агенты (Cursor CLI, чат портала, фоновые раннеры) обязаны **ознакомиться с этим документом перед началом работы над новым продуктом** и **перечитать при явном запросе пользователя**, например: «читай википедию shectory».

## Профили приложений

- **Портальные и пользовательские приложения** — по умолчанию: **welcome-экран с логином**, **единый каталог пользователей** портала Shectory (не изобретать отдельную регистрацию без согласования).
- **Внутренние и сервисные интерфейсы** (например, админка syslog, технические дашборды) могут **не включать полноценный welcome с логином**, если это зафиксировано в архитектуре проекта и не противоречит требованиям безопасности. При сомнениях — уточнять у пользователя и сверяться с карточкой проекта.

## Инфраструктура и данные

- **Хостинг приложений и окружений** — согласованный **hoster** (см. внутренние регламенты деплоя и `scripts/deploy-*`).
- **Базы данных** — на **hoster**; доступ к БД в коде приложений — **только через Prisma** (схема в репозитории, `prisma/schema.prisma`), без произвольного SQL в прикладном коде, кроме миграций и осознанных исключений с документированием.
- Не раздувать `process.env` и аргументы процессов при вызове CLI: длинные промпты передавать через **stdin**, а не в argv.

## Коммиты и деплой

- **Коммиты** — осмысленные сообщения, атомарные изменения; не коммитить секреты и локальные `.env` с ключами.
- **Деплой** — через принятые в репозитории скрипты (например `scripts/deploy-shectory-portal.sh` и аналоги для сервисов). После значимых изменений — проверка работоспособности на целевом окружении.

## Карточка проекта и актуализация

- Поддерживать в актуальном состоянии **описание проекта**, **технологический стек**, **визуализацию архитектуры** (диаграммы, ссылки на артефакты) — в данных портала и/или в репозитории, как принято для конкретного проекта.
- При изменении стека или границ системы — обновить регистр проектов и связанные артефакты welcome/handoff, если они используются.

## Агенты и UI Shectory

- Ответы пользователю в чате портала — **на русском** (кроме имён файлов, команд, идентификаторов API и кода).
- Команды shell в UI выполняются **только после подтверждения** пользователя; выводить блоки `<<<SHELL_COMMAND>>>…<<</SHELL_COMMAND>>>` и маркер ожидания ответа, как описано в системных инструкциях раннера.
- Любой новый продукт или крупная фича: **сначала** — контекст из этой Wikipedia + карточка проекта + актуальные поля тикета/сессии.

## Версия

Документ живой; при изменении стандартов Shectory обновляйте этот файл и кратко отражайте суть в changelog/описании PR при необходимости.
