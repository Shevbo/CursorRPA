# Shectory Welcome/Login Standard (прикладные проекты)

Единый стандарт для экрана входа прикладного проекта в экосистеме Shectory.

## Обязательная композиция экрана

- **Большой информационный фрейм** (визуализация/открытая информация проекта) — кастомный HTML/CSS у каждого проекта.
- **Слева сверху**: логотип Shectory (пока заглушка допустима).
- **Справа сверху**: логотип прикладного проекта (пока заглушка допустима) + версии модулей.
- **Унифицированная область логина** (общий UX и базовая структура форм).

## Требования к унифицированной области логина

- Вход по учётной записи проекта (email/password или эквивалент).
- Понятные статусы ошибок (неверный логин/пароль, блокировка, недоступность сервиса).
- Безопасная сессия (HttpOnly cookie / secure policy по стеку проекта).
- Ссылка/доступ к политике безопасности и контактам поддержки.

## Интеграция с единым сервисом каталога пользователей

Экран входа обязан использовать стандартизованный каталог пользователей/ролей:
- роли минимум `user` и `admin`,
- управление ролями только из защищённой админ-зоны,
- запрет самодемоута admin.

См. также: `docs/unified-auth-users-rbac-ru.md`.

## Для агентов (обязательное правило)

При изменениях welcome/login:
- соблюдать эту композицию экрана,
- не ломать унифицированную область логина,
- не удалять блок логотипов и версий,
- не менять security-поведение авторизации в сторону ослабления.
